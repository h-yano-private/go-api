# タスク管理アプリ 基本設計書
## 1.システム概要
本アプリは、ユーザーの日々のタスクを視覚的に管理するためのwebアプリケーションである。
フロントエンドとバックエンドを分離したモダンなアーキテクチャを採用する

## 2.要件定義(機能一覧)
システムが満たすべき基本要件は以下のとおりとする。

| 機能ID | 機能名 | 説明 | 優先度 |
| :--- | :--- | :--- | :--- |
| F-01 | タスク一覧表示 | 登録されているすべてのタスクを画面にリスト表示する。 | 必須 |
| F-02 | タスク新規登録 | 入力フォームから新しいタスクを追加できる。 | 必須 |
| F-03 | タスク編集 | 既存のタスクのタイトルを修正できる。 | 必須 |
| F-04 | タスク削除 | 不要になった、または完了したタスクを削除できる。 | 必須 |
| F-05 | 完了チェックボックス | タスクの完了状態を切り替えられる。完了タスクは打ち消し線で表示される。 | 必須 |
| F-06 | タスク期限 | タスクに期限を設定できる。期限が近い（3日以内）タスクは赤で表示される。 | 必須 |

## 3. 画面設計（UIレイアウト）
画面は1画面（シングルページ）構成とし、以下の要素を配置する。
```
+------------------------------------------+
|        📋 マイ・タスク管理                 |
+------------------------------------------+
| [ タスクを入力...    ] [日付選択] [追加]  | <- (A) 入力エリア
+------------------------------------------+
|                                          |
|  ☐ タスクA               [編集] [削除]    | <- (B) タスクリストエリア
|    期限: 2026-07-20                      |
|                                          |
|  ☑ タスクB（完了）       [編集] [削除]    |
|    期限: 2026-07-14（赤で表示）          |
|                                          |
+------------------------------------------+
```

### 各エリアの振る舞い（UIインタラクション）
1. **(A) 入力エリア**
   - テキストボックスに文字を入力し、期限をdate pickerで選択して、「追加」ボタンを押すとタスクが登録される。
   - 期限は任意入力可能（未入力でも登録可）
   - 未入力の状態で「追加」を押した場合は、エラー（バリデーション）とする。

2. **(B) タスクリストエリア**
   - 登録されたタスクが上から順に並ぶ。
   - 各タスクの左にチェックボックスがあり、クリックでタスクの完了/未完了を切り替える。
   - 完了したタスクは打ち消し線で表示され、少し薄くなる。
   - 期限が設定されている場合、タスク名の下に「期限: YYYY-MM-DD」形式で表示される。
   - 期限が3日以内（または本日が期限）の場合は赤色で表示される。
   - 「編集」ボタンを押すと、そのタスクのタイトルを変更できる。
   - 「削除」ボタンを押すと、確認のダイアログなしで即座にそのタスクが画面から消去される。

## 4. データモデル

### Task構造体
```go
type Subtask struct {
    ID        int    `json:"id"`          // サブタスクの一意なID
    Title     string `json:"title"`       // サブタスクのタイトル
    Completed bool   `json:"completed"`   // サブタスクの完了状態
}

type Task struct {
    ID        int       `json:"id"`          // タスクの一意なID
    Title     string    `json:"title"`       // タスクのタイトル
    Completed bool      `json:"completed"`   // 完了状態（true=完了、false=未完了）
    Deadline  string    `json:"deadline"`    // 期限（YYYY-MM-DD形式、任意）
    Subtasks  []Subtask `json:"subtasks"`   // 小項目タスク一覧
}
```

## 5. 画面とバックエンドの連動（データフロー）
ユーザーの操作に応じて、フロントエンドは裏側のバックエンドAPIと以下のように連動する。

1. **初期表示時**
   - 画面がロードされたら、自動的にバックエンドからタスク一覧を取得して（B）に描画する。

2. **追加ボタンクリック時**
   - 入力されたテキストと期限をバックエンドに送信し、成功データが返ってきたらリスト（B）の末尾に動的に追加する。

3. **削除ボタンクリック時**
   - 対象タスクのIDをバックエンドに送信し、通信成功後に画面のリストから対象の行を削除する。

4. **チェックボックスクリック時**
   - タスクの完了状態（Completed）をバックエンドに送信し、リストを再表示する。

5. **編集ボタンクリック時**
   - プロンプトで新しいタイトルを入力させ、バックエンドに送信して更新する。
